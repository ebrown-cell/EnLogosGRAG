EnLogosGRAG — quickstart

A local web app for cataloging a corpus of vendor documents. Extracts metadata, vendors, and document types without AI, using two stages of rule-based extraction. Free, deterministic, editable in seconds.

First-time setup

  1. Source listing → Choose folder… Pick the folder whose immediate subfolders are vendor names (e.g. …\Vendors\). The server uses an existing basic_listing.txt if present, or generates one by walking the folder.
  2. Actions → Ingest → Full Process. One click runs the whole pipeline: vendors ingest → files ingest → classify by filename → extract PDF text → classify by content. Each stage is idempotent; safe to re-run.
  3. Charts page reflects the result. Use the preset shortcuts (Classified only, Unclassified PDFs, etc.) to focus on what you want to see.

What the two classifiers do

  • Filename rules (classifier.yaml) match against the file's name, parent folder, full path, and extension. Fast, runs against every file. Catches obvious cases like "…Installation Manual.pdf".
  • Content rules (content_classifier.yaml) match against the extracted PDF text — first-page anchors and full-body keywords. Only runs against documents the filename rules left unclassified or low-confidence.

Edit either YAML file and click the corresponding Classify button — rules reload from disk on every run.

Browsing the data

  • Tables in the sidebar — vendors, document_types, files, documents, document_extracts. Click a table to view its rows.
  • Filter row above the table accepts text, file-type, document-type, classified/unclassified, and confidence-threshold filters. Multi-select for file/document type. Column widths drag-resize and persist.
  • Click a filename or path to open the file in its OS default app (PDF in Acrobat, etc.).
  • Click "view" in the extract column to read a document's extracted text + PDF metadata.

Charts and drilldown

  • Six pies: Classified vs unclassified · By confidence · Top ten filetypes · Other filetypes · By document type · Other document types.
  • Click a slice to compose a filter and re-render the chart page scoped to that subset. Breadcrumb at the top tracks the chain.
  • "View these documents →" jumps from a filtered chart to the documents table with the same filters.
  • When the filter chain narrows so far that no pie has anything to chart, you're auto-routed to the documents table.

Filesystem cache

PDF text extraction is the slow stage. To survive DB wipes, every successful extract is also written to a sibling folder named <Vendors>_text/ as a JSON file mirroring the source tree. On subsequent runs the server reads the JSON cache instead of re-running pdfjs. Use Purge → Purge text backups to clear it.

Editing the rules

Both YAML files live next to the server source. Each rule has id, document_type, confidence (high/medium/low), match (name/parent/path/file_type for filename rules; first_page/extract for content rules), and pattern (a JS regex, case-insensitive). First match wins — put specific patterns above general ones.
